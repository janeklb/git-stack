# Releasing

`git-stack` releases are cut from tags in this repository. The Homebrew formula lives in `janeklb/homebrew-tap`, so a release here and the tap update there are separate repository changes tied together by release metadata.

## Release flow

1. Create and push a `v*` tag in this repository.
2. The tag-triggered release workflow runs tests and publishes the GitHub release assets.
3. The release workflow also computes the immutable source archive URL and SHA256 needed by Homebrew and uploads them as `homebrew-release.json` on the GitHub release.
4. That metadata is used to open or reuse a release-specific issue in `janeklb/homebrew-tap` with the exact inputs needed for the tap update.

## Homebrew expectations

- The tap formula should build from the immutable tagged source archive for the release version.
- The formula should install shell completion files generated by the packaged binary.
- The tap update should happen via a reviewable PR in `janeklb/homebrew-tap`, not by pushing directly to `main`.

## Maintainer notes

- Cross-repo automation needs a `HOMEBREW_TAP_TOKEN` secret with access to `janeklb/homebrew-tap`.
- If the tap-side automation is unavailable, the release metadata still gives a single source of truth for the required formula update.
- The release metadata is generated by `scripts/release-homebrew-metadata.sh`.
- The tap handoff issue is created by `scripts/open-homebrew-tap-issue.sh`.
